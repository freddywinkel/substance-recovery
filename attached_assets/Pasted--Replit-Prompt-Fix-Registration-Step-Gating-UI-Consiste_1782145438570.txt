In all tracker wizards, users can currently click "Next" without answering required questions on certain steps. Add validation so **EVERY required field on a step must be answered before the Next button is enabled.**

### 1a. `src/pages/CravingTracker.tsx`

**Find `canProceed` (around line 271):**
```typescript
  const canProceed = useMemo(() => {
    switch (step) {
      case "onset":   return draft.onsetType !== "" && (draft.onsetType !== "Other" || draft.onsetOther.trim() !== "");

      case "action":  return draft.chosenAction !== "";
      case "outcome": return draft.useOutcome !== "";
      default:        return true;
    }
  }, [...]);
```

**Replace with:**
```typescript
  const canProceed = useMemo(() => {
    switch (step) {
      case "onset":   return draft.onsetType !== "" && (draft.onsetType !== "Other" || draft.onsetOther.trim() !== "");
      case "trigger": return draft.situationPresets.length > 0 && draft.buildupDuration !== "" && (!draft.situationPresets.includes("Other") || draft.triggerOther.trim() !== "");
      case "inner":   return draft.emotions.length > 0 || draft.thoughtPresets.length > 0 || draft.thoughtFreeText.trim() !== "";

      case "action":  return draft.chosenAction !== "";
```

**Why:** The `trigger` step was only checking `buildupDuration` but NOT `situationPresets` (the main trigger selection). The `inner` and `substance` steps had NO validation at all.
