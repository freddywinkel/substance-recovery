# Crisisdienst & Noodcontacten — Prompt voor Replit AI

Voeg crisisdienst- en noodcontact-functionaliteit toe aan de app. Alleen in **Settings** en **Help** tab. **Niet** op het homescreen. Geen redesign.

---

## 1. DATA MODEL (src/db/index.ts)

Voeg toe aan de bestaande interfaces, **niet** bestaande wijzigen:

```typescript
export interface CrisisService {
  id: string;
  name: string;
  number: string;
  isCustom: boolean;
}

export interface EmergencyContact {
  id: string;
  name: string;
  relationship: string;
  phone: string;
}
```

Voeg toe aan `SettingsData` interface:
```typescript
export interface SettingsData {
  // ... bestaande velden
  crisisService?: CrisisService;
  emergencyContacts?: EmergencyContact[];
}
```

Default waarden bij `initDB` (crisisService = null, emergencyContacts = [])

## 4. SETTINGS PAGINA — CRISISDIENST & NOODCONTACTEN

In `src/pages/Settings.tsx` (of het bestaande settings-scherm), voeg twee nieuwe secties toe **onderaan** de bestaande settings, **boven** de versie-info.

### 4a. Crisisdienst-sectie
- Label: `t("settings.crisisService")` met description `t("settings.crisisService.description")`
- Dropdown/select: toon alle 18 `DEFAULT_CRISIS_SERVICES` + optie `"custom"` met label `t("settings.crisisService.custom")`
- Bij selectie van een default: sla op als `{ id, name, number, isCustom: false }`
  - `name` moet de vertaalde string zijn (via `t(nameKey)`), niet de key zelf
- Bij selectie van `"custom"`: toon twee invoervelden:
  - "Service name" (`t("settings.crisisService.customName")`) — tekst
  - "Phone number" (`t("settings.crisisService.customNumber")`) — tekst, valideer dat er minstens 3 cijfers in zitten
- Sla custom op als `{ id: "custom", name: (input), number: (input), isCustom: true }`
- Bij wisselen van default naar custom: leegmaken, bij wisselen van custom naar default: velden verbergen

### 4b. Noodcontacten-sectie
- Label: `t("settings.emergencyContacts")` met description `t("settings.emergencyContacts.description")`
- Toon bestaande contacten als kaartjes (naam, relatie, nummer) met verwijder-knop (`t("settings.emergencyContacts.remove")`)
- "Add contact" knop (`t("settings.emergencyContacts.add")`) — toont alleen als er minder dan 3 zijn. Anders toon `t("settings.emergencyContacts.max")` als grijze tekst
- Bij "Add contact": toon 3 invoervelden:
  - Naam (`t("settings.emergencyContacts.name")`) — tekst, verplicht
  - Relatie (`t("settings.emergencyContacts.relationship")`) — tekst, verplicht (geen dropdown, vrij in te vullen)
  - Telefoonnummer (`t("settings.emergencyContacts.phone")`) — tekst, verplicht, minstens 3 cijfers
  - "Opslaan" knop + "Annuleren" knop
- Validatie: alle 3 velden verplicht, telefoonnummer minstens 3 cijfers. Toon geen alert, maar rode border/error text bij invalid
- Sla op in IndexedDB als `EmergencyContact[]` (max 3 items)

## 5. HELP TAB — WEERGAVE

In `src/pages/Help.tsx` (of het bestaande help-scherm), voeg twee nieuwe secties toe **bovenaan**, voor de bestaande FAQ/accordion.

### 5a. Crisisdienst-kaart
- Alleen tonen als `crisisService` is ingesteld (name + number bestaan)
- Titel: `t("help.crisisService.title")` + naam van dienst
- Grote knop: `t("help.crisisService.call")` — linkt naar `tel:{number}`
- Styling: alert/destacated card (bijv. `bg-red-50 border-red-200` in light mode, `bg-red-950/20 border-red-800/30` in dark mode) zodat het opvalt
- Icoon: `Phone` (Lucide) naast titel

### 5b. Noodcontacten-kaart
- Alleen tonen als `emergencyContacts` array niet leeg is
- Titel: `t("help.emergencyContacts.title")` + `Phone` icoon
- Per contact: kaartje met naam, relatie, nummer
- Per contact: "Bellen" knop (`t("help.emergencyContacts.call")`) → `tel:{phone}`
- Max 3 contacten, verticaal gestapeld
- Styling: subtiele kaarten, `bg-card border border-border`

- Gebruik bestaande UI patterns (Card, Button, Input, Label componenten)
