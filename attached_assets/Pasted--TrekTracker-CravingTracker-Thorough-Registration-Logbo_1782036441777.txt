# TrekTracker & CravingTracker — Thorough Registration, Logbook & Stats

Maak de registratie van zowel **TrekTracker** (actief) als **CravingTracker** (passief) grondig en statistisch relevant. Voeg ontbrekende velden toe (emoties, intensiteit, lichamelijke signalen, uitkomst, locatie) zodat de logbook volledig is en de statistieken betekenisvolle inzichten geven.

---

## 1. DATA MODEL (src/db/index.ts)

Voeg toe aan `CravingLog` interface (bestaande velden blijven ongewijzigd):

```typescript
export interface CravingLog {
  id: string;
  timestamp: number;
  status: "draft" | "completed";
  cravingType: "active" | "passive";
  // bestaande velden:
  type?: string;
  planningStage?: string;
  need?: string;
  action?: string;
  confidence?: number;
  onset?: string;
  trigger?: string;
  // NIEUW — voor beide trackers:
  intensity?: number;            // 1-10
  emotions?: string[];           // multi-select
  physicalSignals?: string[];    // multi-select
  location?: string;             // tekst, optioneel
  outcome?: "used" | "not_used" | "unsure"; // alleen bij completed
  // NIEUW — alleen TrekTracker (actief):
  triggerNote?: string;          // "Wat zette dit in gang?"
  // NIEUW — alleen CravingTracker (passief):
  thoughts?: string[];           // multi-select cognitieve verstoringen
}
```

- `intensity`: 1-10, verplicht bij elke completed log
- `emotions`: string array, max 3 selecties (voorkom overload)
- `physicalSignals`: string array, max 3 selecties
- `outcome`: verplicht bij completed — "used" (toegegeven), "not_used" (weerstaan), "unsure" (weet nog niet)
- `thoughts`: alleen voor passive cravings, max 2 selecties
- `location`: optioneel, tekstveld
- `triggerNote`: optioneel, tekstveld, alleen actief

## 3. TREKTRACKER (ACTIEF) — 5 STAPPEN

Behoud alle bestaande stappen en opties. Voeg nieuwe velden toe aan bestaande of nieuwe stappen. Gebruik **multi-select chips** voor emoties/lichaam/gedachten — snel te tappen, max 3 (of 2) selecties.

### Stap 1: Type + Intensiteit + Locatie
- **Type** (bestaand): "Planning", "Actief zoeken", "Ritueel", etc. — single select
- **Intensiteit** (NIEUW): 1-10 slider, verplicht. Toon labels: 1=Mild, 5=Matig, 10=Overweldigend. Kleur progressie: groen → geel → oranje → rood
- **Locatie** (NIEUW): optioneel tekstveld, onder intensiteit

### Stap 2: Planning + Emoties + Lichamelijk
- **Planning stage** (bestaand): "Alleen erover nadenken", "Onderweg", etc.
- **Emoties** (NIEUW): multi-select chips, max 3. 11 opties. Toon geselecteerde chips als `bg-primary/20 text-primary` 
- **Lichamelijke signalen** (NIEUW): multi-select chips, max 3. 8 opties. Zelfde chip styling

### Stap 3: Need + Trigger Note + Gedachten
- **Need** (bestaand): "Verlichting", "Opwinding", "Troost", etc.
- **Trigger Note** (NIEUW): optioneel textarea. "Wat zette dit in gang?"
- **Gedachten** (NIEUW): multi-select chips, max 2. 8 opties. Alleen voor actief — planning cravings gaan gepaard met specifieke rationales

### Stap 4: Action
- **Action** (bestaand): "Toegang verwijderen", "Locatie veranderen", etc. — single select

### Stap 5: Uitkomst + Confidence
- **Uitkomst** (NIEUW): 3 grote buttons naast elkaar (of gestapeld):
  - 🔴 "Ik heb gebruikt" (bg-red-100 border-red-300 text-red-800 / dark: bg-red-950/30 border-red-800/40 text-red-300)
  - 🟢 "Ik heb niet gebruikt" (bg-green-100 border-green-300 text-green-800 / dark: bg-green-950/30 border-green-800/40 text-green-300)
  - ⚪ "Weet ik nog niet zeker" (bg-muted border-border text-muted-foreground)
- **Confidence** (bestaand): "Hoe zeker ben je dat je niet gaat gebruiken?" slider 0-100
  - **NB**: als uitkomst = "used", pas confidence label aan naar "Hoe zeker ben je dat je nu kunt stoppen?" (of equivalent)

## 6. INTENSITEIT SLIDER

1-10 slider met visuele feedback:

```tsx
<div className="space-y-4">
  <div className="flex justify-between text-xs text-muted-foreground">
    <span>{t("intensity.mild")} (1)</span>
    <span>{t("intensity.overwhelming")} (10)</span>
  </div>
  <input
    type="range"
    min={1}
    max={10}
    value={intensity || 5}
    onChange={(e) => setIntensity(Number(e.target.value))}
    className="w-full accent-primary"
  />
  <div className="text-center">
    <span className="text-2xl font-bold text-primary">{intensity}</span>
    <span className="text-sm text-muted-foreground ml-2">
      {intensity <= 3 ? t("intensity.mild") : 
       intensity <= 6 ? t("intensity.moderate") : 
       intensity <= 8 ? t("intensity.strong") : t("intensity.overwhelming")}
    </span>
  </div>
</div>
```

- Verplicht: Next knop disabled tot intensity > 0
- Default waarde: 5 (midden)

## 8. STATISTIEKEN / VOORTGANG (src/pages/Progress.tsx of src/lib/analytics.ts)

Voeg nieuwe statistieken en charts toe. Behoud alle bestaande stats.

### 8a. CravingStats interface (src/lib/analytics.ts)

```typescript
export interface CravingStats {
  // ... bestaande velden
  // NIEUW:
  totalCompleted: number;
  outcomeUsed: number;
  outcomeNotUsed: number;
  outcomeUnsure: number;
  successRate: number;           // percentage not_used / totalCompleted
  avgIntensity: number;          // gemiddelde 1-10
  avgIntensityActive: number;
  avgIntensityPassive: number;
  topEmotions: { name: string; count: number }[];
  topPhysical: { name: string; count: number }[];
  topThoughts: { name: string; count: number }[];
  intensityByWeek: { week: string; avg: number; count: number }[];
  outcomeByAction: { action: string; used: number; notUsed: number; rate: number }[];
  outcomeByNeed: { need: string; used: number; notUsed: number; rate: number }[];
  outcomeByEmotion: { emotion: string; used: number; notUsed: number; rate: number }[];
}
```

### 8b. Analytics functies (src/lib/analytics.ts)

Voeg helper toe die alle bovenstaande berekent uit `CravingLog[]`. Logica:

- `totalCompleted`: logs met `status === "completed"`
- `outcomeUsed/NotUsed/Unsure`: count per `outcome` veld
- `successRate`: `(outcomeNotUsed / totalCompleted) * 100`, round to 1 decimal
- `avgIntensity`: average van alle `intensity` velden (alleen completed)
- `topEmotions`: flatten alle `emotions` arrays, count occurrences, sort desc, top 5
- `topPhysical`: zelfde voor `physicalSignals`
- `topThoughts`: zelfde voor `thoughts`
- `intensityByWeek`: groepeer per week (ISO week), gemiddelde intensity, count
- `outcomeByAction`: per `action` value, count used vs notUsed, bereken success rate
- `outcomeByNeed`: zelfde per `need`
- `outcomeByEmotion`: zelfde per `emotion` (flatten emotions arrays)

## 9. DARK MODE

Alle nieuwe UI elementen moeten dark mode compatible zijn:
- Chips: `dark:bg-muted dark:border-border dark:text-muted-foreground` + selected `dark:bg-primary/20 dark:border-primary/40 dark:text-primary`

- Uitkomst buttons: gebruik `dark:` prefixen voor rood/groen/grijs varianten
- Progress bars: gebruik `bg-primary` voor fill, `dark:bg-muted` voor track
- Logbook kaarten: `dark:bg-card dark:border-border`
